Database ideas
---------------
1. Arrays as tables.
2. Work flows (Stored Procedures) can be deployed as Spark/Flink jobs.
3. database + kafka(streaming) - easy to get data in and out (json, csv, avro, parquet, orc)

Chimera

Incoming Features
-----------------
1. BLOBs in separate file - MEDIUM
2. indexed fields vs payload (event sourcing) - MEDIUM
3. allocatable blocks for TEXT string types (all variable sized types) - MEDIUM/LARGE
4. Circular tables - MEDIUM
5. implement: onDelete/onInsert - SMALL
    TableFile.onRowUpdate { (row:  BinaryRow) => ... }
6. To support large gaps in RowIDs, we could use offset translation to localized offsets/files like partition device (hashmap)
7. Add support for Arrays - MEDIUM [see https://cloud.google.com/bigquery/docs/reference/standard-sql/array_functions]
    a. SELECT value FROM STRING_SPLIT('Roberto,Gail,Dylan', ',');
    b. SELECT Array(1, 2, 3) AS numbers;
    c. SELECT Array("apple", "pear", "orange") AS fruit;
    d. SELECT Array(true, false, true) AS booleans;
8. Add support for variables as tables - MEDIUM
    DECLARE @myTableVariable TABLE (id INT, name varchar(20))
    INSERT INTO @myTableVariable VALUES(1,'Roberto'),(2,'Gail'),(3,'Dylan')
    SELECT * FROM @myTableVariable
9. Add support for RowID (__id) in the WHERE clause - SMALL
    SELECT count FROM 'public'.'stocks' WHERE __id BETWEEN 1 AND 100;
    UPDATE 'public'.'stocks' SET count = 1200 WHERE __id = 100;
    UPDATE 'public'.'stocks' SET comments = 'N/A' WHERE __id BETWEEN 0 AND 25;
10. Column-wise SSE-like functions
